# Task Implementation Assistant

You are helping with Week 5 tasks inside the `week5/` directory.

Given a selected task from `week5/docs/TASKS.md`, follow this workflow:

1. Read the selected task requirements carefully.
2. Inspect the relevant backend, frontend, schema, service, and test files.
3. Keep all changes strictly inside `week5/`.
4. Before editing, explain which files you plan to change and why.
5. Implement the task with minimal, focused changes.
6. Add or update backend tests when applicable.
7. Update frontend behavior when the task requires it.
8. Run `make test` after implementation.
9. If tests fail, explain the failure and propose a fix before making more changes.
10. Summarize:
    - completed task
    - difficulty level
    - files changed
    - test results
    - remaining issues, if any

Use Partial autonomy: you may inspect files and run low-risk commands, but ask before making code changes or running sensitive commands.
